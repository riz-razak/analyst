1. The Problem: $0.84/Driver Maps API Cost

The Core Issue

At 120 rides/driver/month and Google Maps Routes API pricing of ~$0.007/ride, the Maps API alone costs $0.84/driver/month. For 5,000 drivers, that's $4,200/month on maps alone — roughly 76% of total ops cost at the $5,500 budget. This is unsustainable and represents the single largest variable cost in the WarenYan stack.

Why This Matters for the Business Model

WarenYan's entire competitive advantage is zero commission. But if the platform's per-driver cost is dominated by a single vendor (Google), the company has no pricing power and no path to profitability at scale. Every additional driver adds ~$0.84/month in Maps API cost before a single rupee of subscription revenue is earned. The break-even equation collapses if we can't control this cost.

The $0.007/ride figure comes from a typical ride requiring: 1 Routes API call (route planning), 0.5 Geocoding calls (address resolution), and periodic Distance Matrix calls for ETA estimation. At Google's standard pricing of $5-10/1K requests, the per-ride cost ranges from $0.005 to $0.010 depending on feature usage.

4. Regional Alternatives: How India Solved This

Namma Yatri — The Blueprint 130M+ rides on $5M total spend

Stack: OpenStreetMap data + OSRM (Open Source Routing Machine) + custom frontend

Protocol: ONDC/Beckn (open network protocol for mobility)

Result: 700K+ drivers, 13M+ riders in Bengaluru & Chennai. Fully open-source on GitHub. Per-ride mapping cost approaches $0.001 at scale — 90%+ savings vs Google.

Key insight: They proved that OSM + OSRM handles real-world Indian urban routing at massive scale. If it works in Bengaluru traffic, it works in Colombo.

Mappls (MapMyIndia) India-Specific

India's homegrown mapping platform. Deep tier-2/3 city coverage. Used by government and enterprise. Pricing not public but significantly below Google for Indian deployments. Not available for Sri Lanka.

BharatTaxi Government-Backed

Zero-commission model (like WarenYan). Likely uses open-source mapping. Focuses on tier-2/3 cities. Validates the model but doesn't export technology.

For WarenYan's Colombo beachhead strategy, the data is sufficient for launch. The file size is just 135MB — tiny enough to pre-process and serve from a single regional server.

Local Infrastructure Risks

Risk: Dialog/Mobitel don't offer mapping APIs

Unlike India (where Jio Maps and Mappls/MapMyIndia provide alternatives), Sri Lanka has no local mapping provider. This means the open-source route is the only alternative to Google/Mapbox. No commercial fallback exists within the country.

Risk: Internet reliability for self-hosted services

If OSRM/Nominatim are hosted in Singapore or Mumbai (nearest AWS/DO regions), Sri Lanka's international bandwidth matters. Average latency to Singapore: ~40ms. For routing queries this is acceptable, but map tile loading may feel sluggish without aggressive caching. Mitigation: Use Cloudflare R2 for tile CDN (Colombo PoP exists) and pre-cache common routes.

Verdict: The Namma Yatri Precedent Proves the Path

Namma Yatri operates at 130M rides on $5M total spend. Their mapping cost per ride is ~$0.001. If WarenYan achieves even 10x their per-ride cost ($0.01), it would still be 37% cheaper than Google at full price. The open-source stack is not a compromise — it's a competitive advantage. The investor narrative shifts from "we depend on Google" to "we control our own infrastructure like Namma Yatri."

9. Recommendation & Implementation Path

Phase 1: MVP (Weeks 1-4) — Mapbox + OSRM

Cost: ~$200/month. Use Mapbox free tier (50K map loads) for rendering and OSRM (Docker container) for routing. Fastest path to market. Mapbox gives better developer experience than raw MapLibre for MVP speed.

Phase 2: Independence (Weeks 5-12) — Full Self-Hosted

Cost: ~$400/month. Replace Mapbox rendering with MapLibre + Protomaps on Cloudflare R2. Add Nominatim for geocoding. Full control, zero vendor dependency. This is the Namma Yatri architecture.

Phase 3: Optimization (Month 3+) — Data Enhancement

Cost: ~$350/month (reduced through optimization). Contribute to OpenStreetMap for Colombo-specific improvements. Cache frequently-used routes. Build historical ETA models from actual ride data (replaces need for Google traffic data). This is the long game — WarenYan's own data becomes the competitive moat.
