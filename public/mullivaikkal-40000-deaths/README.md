# Mullivaikkal 40,000 Claim SEO Evidence Package

This folder is the portable publication package for the research-to-correction workflow.

Keep only these elements:

- `/readme` (this document): package orientation and what is included
- `/methods`: replication and stress-test process files
- `/data`: source and SEO tables used for correction work

## Core factual position

The strongest source chain does not support `40,000 deaths in Mullivaikkal alone` as stated.

What is supported is a cautious final-phase Vanni civilian death estimate context with caveats on method, category, and geography. The package is organized so every correction claim can be tied back to this structure.

## Crawl Status Caveat

The `1,000`-link layer is a work-in-progress accounting and prioritisation frame. The package includes a reviewed top-100 seed capture, live crawl batches, and an inspected URL register, but it is not a completed live top-1,000 crawl.

The next URL task is deliberately narrower: map candidate URLs, fetch them, dedupe them, and record status. Claim assessment, scoring, screenshots, and archive checks should happen only after that URL sheet exists.
